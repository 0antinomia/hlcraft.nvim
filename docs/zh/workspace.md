# 工作台

[索引](./index.md) | [English](../en/workspace.md)

hlcraft 工作台是搜索、查看、编辑、预览和保存高亮 override 的交互界面。

打开方式：

```vim
:Hlcraft
```

## 布局

工作台围绕三类操作组织：

- 搜索：按名称或颜色缩小高亮组列表。
- 查看：读取当前高亮组的解析值、来源、链接和 override 状态。
- 编辑：修改分组、颜色、样式、blend 或动态颜色。

## 搜索

顶部区域有两个输入：

- `name`：按高亮组名称做大小写不敏感的子串匹配。
- `color`：按颜色相似度搜索。

两个过滤条件可以组合使用。

颜色查询支持：

- `#RRGGBB`
- `NONE`

`NONE` 会匹配解析后颜色未设置的高亮组。默认检查 `fg` 和 `bg`；启用 `search.include_sp` 后也会检查 `sp`。

## 详情视图

详情视图展示当前高亮组中 hlcraft 能处理的信息：

- 解析后的 `fg`、`bg`、`sp`。
- `bold`、`italic`、underline 相关样式。
- blend 值。
- link 链路和最终解析目标。
- 草稿和持久化 override 状态。
- 持久化分组。

详情视图适合反复微调：修改一个字段，立即看到效果，满意后再保存。

## 编辑器

每类字段都有独立编辑器：

- 分组编辑器：选择或创建保存 override 的 TOML section。
- 颜色编辑器：设置、清除、预览颜色，或把颜色通道切换为动态模式。
- 样式编辑器：切换布尔样式属性。
- Blend 编辑器：设置或清除 blend。
- 动态编辑器：调整 preset、duration、phase，以及当前动态通道的原始 JSON。

非法输入会在修改草稿状态前被拒绝。

## 预览按键

预览按键用于短暂闪烁当前选中的高亮组，帮助你在真实 UI 中定位它。

默认配置：

```lua
keymaps = {
  preview = {
    lhs = 'z',
    mode = 'n',
  },
}
```

这个映射只在工作台打开期间存在。它在这段时间里是全局普通模式映射，所以切换到其他窗口后仍然可用。

## 保存和放弃

草稿 override 会立即应用。只有保存后，它才会写入 TOML 并跨会话保留。

当你希望当前高亮组的草稿结果在重启或 colorscheme 变化后继续存在，就执行保存。想回到持久化状态时，使用放弃或恢复操作。

## TUI 边界

hlcraft 运行在 Neovim 终端 UI 里。它可以预览高亮组级别的变化和动态色块，但不能做像素级动画、跨字符空间渐变或逐字符运动效果。动态颜色的本质是按时间更新高亮组的颜色值。
